## 2026-05-11 — Session 10 of 10 (Tuning + Cinematics Implementation)
- S10 #attract-impulse-to-lerp-when-user-feels-pendulum: User reported "stupid magnet slowly swinging back and forward" — described physics as "stupid" rather than describing an outcome. Replaced impulse-on-prevPos (k = ATTRACT_STRENGTH / dist, pushed against prevPos under verlet damping 0.998 = damped pendulum) with direct position-lerp: `pos += (cursor - pos) * rate; prevPos = oldPos`. Snappy follow, no overshoot, residual velocity = lerp delta. **Lesson: when the user describes physics as "stupid" or names the artifact (swinging) rather than the desired outcome, switch from physics simulation to direct UI lerp. They want tracking, not attraction.**
- S10 #bfs-hop-map-as-effect-payload: STRUCTURE_GROW carries precomputed BFS hop maps (`Map<PrimitiveId, hop>` + `Map<BondId, hop>`) at emit time. Renderer looks up CURRENT primitive/bond positions per frame; severed-mid-effect IDs silently skipped via `Map.get → undefined`. Maps are fine because effects are write-only telemetry (not persisted). **Lesson: effects can carry arbitrary structured payloads — precompute at emit time, look up live state at draw time. Robust against world mutation during the effect's lifetime; cheaper than recomputing BFS per frame.**
- S10 #verlet-impulse-via-prevpos-offset: To impulse a primitive toward a target via verlet, push `prevPos` AWAY from target by `magnitude * unit_vector(target - prim)`. Next-step velocity = `(pos - prevPos)` points TOWARD target. Single line, no force integration, no dt coupling. Decays via VELOCITY_DAMPING. **Lesson: in position-based dynamics, instantaneous velocity is a function of (pos - prevPos) — perturbing prevPos is the cheapest impulse primitive. Magnitude in pixels = directly visible. Safety = compute strain ratio at LOW-tier worst case (1.2 px / 60 px bond = 2%, well under 2.0× break).**
- S10 #cinematic-physics-half-stays-unconditional: P5 cinematics toggle gates EMISSIONS of STRUCTURE_GROW/MERGE/SCORE_TIER. The P3 verlet impulse on the candidate component is NOT gated — user picked "real verlet impulse" over "visual-only", so the physics event is a designed mechanic, not cinematic decoration. **Lesson: distinguish cinematic-as-decoration from cinematic-as-state-mutation. Toggle suppresses the former; the latter is a designed behavior that stays on regardless of visual preference.**
- S10 #test-via-pure-helper-export: AttractDrag impulse logic refactored into `stepAttractLerp(pos, prevPos, cursor, rate)` — pure function exported from controls.ts. Lets unit tests verify lerp math (one-step closure, prevPos contract, 4-step geometric closure, 50-step bounded approach) without spinning up a Pixi Application + DOM mock. **Lesson: when an inline 5-line algorithm needs unit tests but the surrounding class is hard to instantiate, extract the algorithm as a pure exported function. Tiny API surface increase, large testability win, plus named operation > inline mutation.**
- S10 #cross-cutting-gate-deferred-to-last-priority: P5 (cinematics toggle) was the LAST priority in the batch despite touching emission sites in P2/P3/P4. Each emission site shipped UNCONDITIONAL first; P5 added `if (world.cinematicsEnabled)` wraps as a post-hoc gate. Commit log reads cleanly per priority. **Lesson: when a batch has N implementation priorities + 1 cross-cutting gate, order the gate LAST. Each priority commits unconditional + testable in isolation; the gate priority then wraps each emission site. Beats threading the gate through every priority.**
- S10 #anchor-place-emits-grow-too: P2's STRUCTURE_GROW fires on EVERY PLACE_PRIMITIVE including anchors (no bond). Single-anchor placements emit with `hopByPrimId = {origin: 0}` and empty bond map → renderer flashes only the new prim, no cascade. Natural minimum-event for "the structure has one element." Caught in test failure: session5.test.ts asserted `effects.length === 0` on anchor place; updated to filter by kind. **Lesson: a new ubiquitous emission kind breaks legacy "no effects emitted in case X" assertions. Audit tests asserting on `effects.length` vs `effects.filter(kind).length` before adding cross-cutting emissions.**
- SESSION #playtest-confirmed-bug-fix-as-first-priority: S10 P1 (AttractDrag tuning) was the only S10 priority based on direct-playtest user report.
