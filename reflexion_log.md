## 2026-05-13 — Session 23 (P1 chain rewrite + P2 debug overlay + P3 instr + P4 cursor fix)

- S23 #cursor-equality-bug-in-effect-drain: Both `drainAudioEffects` and `runGodlyMatcher` used `if (effect.tick <= cursor) continue` with cursor advancing to `world.tick` at end of frame. Click-handler dispatches between physics ticks emit BOND_FORMED with tick=current (un-advanced) world.tick, which EQUALS the cursor → silently skipped. Manifested as TWO simultaneous reported regressions (Voltkin trigger never fires + SFX claves/farts silent), both rooted in the same off-by-one. Fix: `<=` → `<`. Tests don't catch this because they don't simulate the click-between-rAF-frames timing. **Lesson: when a cursor's "have-we-processed-this" check uses `<=`, you forbid same-tick re-entry; this only works if effects are GUARANTEED to arrive AFTER the cursor advances. In an event-driven game where input dispatches synchronously between frame ticks, that guarantee doesn't hold. AUDIT all cursor patterns with `<=` against tick-driven event streams across the codebase: anywhere you have a cursor and same-tick events from external input, the check should be `<` not `<=`. This is a recurring class-of-bugs, not a one-off.**

- S23 #debug-overlay-as-root-cause-discovery-instrument: User reported "Voltkin doesn't fire" + "SFX silent" simultaneously. Tests green (432/432), code looked correct on every read, offline repro impossible. Shipped `?debug=1` toggleable on-screen panel (~250 LOC, click-to-copy clipboard) surfacing 7 runtime gates (gameState/isHost/cinematicActive/recipes-registry/cooldown/matcher-will-run/BOND_FORMED-flow) + audio chain wiring + live chain progress + counters. User pasted snapshot at trigger moment. Root cause was OBVIOUS from data: `firedEver:false` despite valid 8/8 chain + `clave calls: total=0` despite matcher seeing 7 BOND_FORMED events → same cursor bug. Without the overlay this would have been 5+ deploy-cycle hunt. **Lesson: when offline tests pass and the bug is "in the production runtime," a permanent (or URL-toggleable) debug overlay that surfaces ALL the gates the code consults is the single highest-ROI diagnostic tool you can ship. Cost: ~250 LOC. Benefit: collapses a multi-hour blind hunt into a 5-min paste-and-fix. Build the overlay framework BEFORE the first user-reported runtime bug, not after, so it's already there when needed.**

- S23 #prime-audit-Δ1-preemptive-vitest-fixture-skipped-an-unnecessary-deploy: PRIME-AUDIT after R1+R2 surfaced "could we write a vitest fixture mirroring user's exact build topology?" — a step Council hadn't explicitly demanded. Δ1 fixture (L-shape SQ4-TR4 + reverse direction) was written + run offline in 30 seconds. PASSED, conclusively ruling out the predicate. Bug-hunt then targeted runtime ONLY, saving an exploratory deploy cycle. **Lesson: PRIME-AUDIT's most valuable function is finding the question Council rubber-stamped past. The discriminating step ("write the user's exact reproduction as a unit test") is often the cheapest highest-confidence step. Always ask in PRIME-AUDIT: "could we reproduce this without deploying?" before scoping diagnostic-deploy work. Saves at least one cycle on average.**

- S23 #scope-amendment-rule-16-fired-twice-in-one-session-with-full-tier-council-each: User reported regressions after Phase 1 ship → full-tier Council deliberation → ship P2 overlay → user pastes data → instrument predicate + ship P3 fallback → still doesn't work → diagnose `<=` cursor → ship P4 fix → works. Two scope amendments in one session, both legitimately requiring Council-tier deliberation. **Lesson: scope amendments arrive at the user's pace, not yours. You may finish executing one Council-deliberated PDR and immediately face a second amendment that demands its own Full-tier cycle. Don't try to roll the new scope into the closing summary or "just one more fix" — formally amend, deliberate, decide whether to defer to next session based on context budget. S23 ran 4 priorities + multiple Full-tier deliberations and stayed GREEN because each was scoped tightly.**

- S23 #ship-cinematic-stamp-as-phase-1-defer-creature-actor-to-phase-2: S22 P4 shipped Voltkin as "cinematic intro + sprite stamp + SEVER cascade" — a working MVP. User upon playtesting: "this is super low quality, I wanted a real creature with AI." ## 2026-05-13 — Session 23 (Micro PDR P1 — Voltkin recipe rewrite to typed chain)

- S23 #continuous-threshold-predicates-are-undiscoverable-without-HUD: S22 P4 shipped the geometric Voltkin predicate (lightning aspect>=2.5 + TV-frame aspect 1.0-1.8 + 200 px centroid adjacency) and tests passed. But user playtest revealed it's basically unfireable in natural play: the play field is 1900+ px wide, players spread structures out, and 200 px adjacency means "almost touching" — invisible without an overlay showing what the predicate is reading. User: "geometric constraint is too vague." Pivot was instant once data came in: replace with discrete typed-prim chain (SQ4-TR4). **Lesson: any recipe predicate built on continuous-parameter thresholds (aspect ratio, adjacency distance, area ratio) needs PAIRED debug HUD shipped in the same session, otherwise the recipe is theoretically-correct but practically-invisible. The HUD isn't a debug nicety, it's a co-requisite. Alternative — and what we did here — is to design predicates around DISCRETE features (primitive types, bond chain length, specific topology) where the player can mentally verify "did I place 4 squares?" without needing the game to show them anything. Discrete predicates self-document via the play action; continuous predicates require external feedback.**

- S23 #bond-graphs-are-bidirectional-the-spec-is-not: User spec said "square x4 followed by triangle x4 — in this order." I wrote a DFS path-finder that walks the bond graph from each Square prim looking for the type sequence. First test run: my "wrong order" test (chain TR-TR-TR-TR-SQ-SQ-SQ-SQ should be null) FAILED because the DFS starting from the SQ end walks back through 4 SQs then 4 TRs and matches. Insight: a bond chain has no canonical start; reading it left-to-right vs right-to-left is the same physical structure. The user spec's "order" describes how they MENTALLY BUILD it; the predicate operates on the resulting STRUCTURE which is symmetric. Bidirectional match is the correct UX. **Lesson: when translating ordered-sequence specs ("A then B") into graph predicates, the order has to be anchored to something physically asymmetric — first-placed timestamp, distance to win-spark, or color polarity — otherwise the graph topology imposes bidirectional read by default. If you don't have an anchor and don't want one, embrace bidirectional and update the spec mental model. The discriminating test cases are interleaved sequences (SQ-TR-SQ-TR — null) and filler bridges (SQ4-Circle-TR4 — null), not direction reversal.**

- S23 #micro-tier-PDR-with-user-explicit-go-skips-Council-cleanly: P1 was a single production file + test rewrite, ~150 LOC each, contracts unchanged. User said "go" explicit. Per global Rule 17 Micro auto-waive on user-path, Council deliberation skipped. Wrote inline PDR in session-state.json session_note (no separate .claude/plans/ file). Total wall time from spec lock to push: ~10 min. **Lesson: Micro tier exists for a reason — when scope is tightly contained (1-2 files, no contract changes, no external state, no integration ripples) AND the user has explicit-go'd, the overhead of full PDR file + Council adds zero quality and significant latency. Micro inline-in-state pattern is correct here. The discipline to NOT escalate is as important as the discipline to escalate when needed. Test for Micro: can I describe scope in 3 bullet points AND list all touched files on one line AND have user say "yes do it" — if all three, Micro.**

The signal "I added a new godly" is just "I added a new file + one import line." Beautiful.**

- S20 P3 #vite-tree-shakes-restructured-modules-to-byte-identical-bundle: Pre-extraction main bundle = 397.65 KB.
