This PDR executes §6 Phase 1a **merged with** the pentagram recipe + ignition from §6 Phase 1b, so the slice is genuinely playable (owner wants to playtest the real "build a pentagram → it comes alive" loop + strict shape-gating, not a debug keypress).

## OBJECTIVE
Ship a working, deterministic, multiplayer-safe tower-defense loop: a player builds **exactly 5 triangles in a closed pentagram** → the structure **comes alive as a visible spawn-zone** (radiating aura) that earns a tiny VP trickle and **emits a "chewer" creature every 15 s** → chewers hop to the **nearest ENEMY connector** and **chew it (5 hits / 5 s)** then move on → the spawner is **destroyable** (remove a triangle OR connect an extra shape → it stops, exactly), and chewers are killable (potato blast). Behind a perf/determinism gate. Chewer ships with a **high-quality procedural pencil-style render + amusing physics-driven hop** (real movement, not a looping image); generated pencil art is the immediate follow-on.

## SCOPE (generalize the Voltkin substrate; reuse > reinvent)
Per `TOWER_DEFENSE_DESIGN.md` §2–§5. Concretely:
- **Types/config:** `SpawnerId` in `types.ts`; de-hardcode `VOLTKIN_ATTACK_*` + `CREATURE_MAX_ACCEL` into per-`CreatureConfig` reads (gated by byte-identical Voltkin regression); add `'chewer'` config (`persistent:true`, `chewHits:5`, `hopSpeedMul:~0.6`); `makeCreature(config,args)` with `makeVoltkinCreature` as thin wrapper. Additive `Creature` fields `sourceSpawnerId`/`chewProgress` (host-only).
- **Creature FSM:** wrap auto-delete (`:135`) + forced-DESPAWN (`:144`) in `if (!config.persistent)` — Voltkin path byte-identical. Chewer stays in ATTACKING for the full `chewHits×CHEW_INTERVAL_TICKS`, increments `chewProgress`, severs only on the 5th hit, no re-seek while `chewProgress>0`, resets only when the bond vanishes. `findNearestBondTarget(world,creature,enemyOnly)` — chewers pass `true` (never eat own structure). Split max-1 cap by population (`sourceSpawnerId` null vs non-null) independently; chewer caps `CHEWER_MAX_GLOBAL=8`/`PER_SPAWNER=2`/`PER_VICTIM=3`.
- **Spawner:** `state/spawners/spawner.ts` (`CreatureSpawner` leaf) + `spawnerLifecycle.ts`; `creatureSpawners:Map`+`nextSpawnerId` on `World`. Host-only tick poll in `main.ts` (modeled on the bomb-dissipate / `pendingCreatureSpawn` tick polls — NOT `game/spawner.ts` wall-clock, NO 6th RNG stream): throttled re-validation (`lastValidatedTick`, every 30t) removes the spawner if `!primitives.has(anchor)` or `!recipeStillSatisfied`; emits a chewer when `tick>=nextSpawnTick && underCaps`.
- **Recipe + ignition:** `godlyRecipes/pentagram.ts` — predicate matches **exactly** 5 triangles forming one closed 5-cycle, anchor component == that shape (extra attached primitive → fails; missing triangle → fails). Split `GodlyRecipe` type for non-cinematic recipes. `godlyOrchestration` dispatches `REGISTER_SPAWNER` **without** touching `activeCinematicPlayerId`/`pendingCinematics`; spawner recipes **excluded from** `godlyFiredThisMatch`, gated per-`(playerId,anchor)` against live `creatureSpawners` (rebuildable after a raid; two players can each have one). Lowest-anchor tie-break.
- **Scoring:** tiny `SPAWNER_INCOME_COMPLEXITY` term in `computeComplexity` (near-zero; never touches `PHASE_1_WIN_SCORE=630`). `SPAWNER_KILL_REWARD` via `gameMode.addScore` (no parallel accrual loop).
- **Counterplay (Phase-1 kill path):** extend `applyPotatoDetonate` to `DESPAWN_CREATURE` chewers in blast radius (owner-agnostic, sorted-`CreatureId`). FFA target-spread via `mix32(creatureId,sourceSpawnerId)` + leader-weight. `botBrain` spawner-sever priority + chewer-avoid.
- **Persistence/wire:** host save round-trips `despawnAtTick`/`chewProgress`/`sourceSpawnerId`/`targetBondId` (SerializedBomb precedent) — wire (`netSnapshot`) STRIPS them; render-trimmed creature wire shape; `creatureSpawners?` additive-optional + nextId-advance + nullish guards; `PROTOCOL_VERSION` 9→10; TD actions in `KNOWN_GAME_ACTION_TYPES_RECORD` only (NOT `CLIENT_INTENT_TYPES`). `teardownSpawners` wired into all 4 teardown sites.
- **Art/VFX (this slice = high-quality procedural):** new `chewerRenderer.ts` — original child's-pencil aesthetic (rough/wobbly strokes, oversized goofy teeth), **amusing physics-driven hop** (squash-stretch + vertical arc keyed off Verlet motion + state, faces target via atan2) — real movement, NOT a looping sprite. Spawn-zone visual: radiating pulsed aura + special-styled bonds on the live anchor component. `CHEW_BITE` host-local effect (clone `drawBombExplode` small ring + graphite dust). `creatureRenderer` stays Voltkin; chewer is its own renderer (per-type atlas generalization deferred to the generated-art pass).
