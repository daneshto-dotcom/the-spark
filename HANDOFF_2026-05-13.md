## VOLTKIN — HOW IT WORKS (cheat-sheet for next session)

**Trigger:** Build a lightning-bolt-like structure (≥3 prims, elongated, aspect ratio ≥2.5) within 200 px of a TV-frame-like structure (≥4 prims, squarish, aspect 1.0-1.8). Triggerer = dominant placerColor in the lightning component.

**Sequence:**
1. BOND_FORMED emitted by placePrimitive on the qualifying bond
2. Host runs `findGodlyMatch` over registered recipes (`voltkin` only in v1)
3. Match found → host dispatches local `GODLY_TRIGGER` + broadcasts GodlyTriggerMsg
4. Reducer sets `world.activeCinematicPlayerId` + `world.currentCinematicEvent` + cooldown + SEVER_BOND cascade on target component bonds
5. `startCinematicIfNeeded` in main.ts ticker observes transition + plays cutsceneOverlay
6. 300 ms fade-in → 4 s mp4 with luma-key shader (white → transparent) → voltkin-voice.ogg at t=3.5 s → zap-sprite crossfade at t=4.0 s on targetPos → 8 s sustained → 300 ms fade-out → GODLY_COMPLETE
7. Codex unlocks (`spark:codex:unlocked:v1` localStorage), Voltkin tile becomes full-color

**Locks:**
- 60s cooldown per player (tick-based, 3600 ticks @ 60Hz)
- Asymmetric input-lock — only triggerer's input is gated during cinematic
- Opponent sees Voltkin-yellow vignette during 8 s sustained window
- Host-only predicate; client never runs detection (anti-desync)
- Codex empty default; unlock only on successful host-validated trigger

**Cooldown via save/replay:** authoritative ticks; UI displays seconds via wall-clock conversion. NetSnapshot apply resets cooldown to null (cooldown is host-authoritative anyway).
